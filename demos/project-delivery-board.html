Project delivery board

Prototype April 28, 2026

Overview

A lightweight project tracking board that focuses on what matters: clear ownership, visible risks, and regular checkpoints. No Gantt charts, no complex dependencies—just clarity.

The Problem

Traditional project management tools are either:

  • Too simple: Sticky notes with no structure
  • Too complex: Enterprise tools requiring dedicated administrators

Teams need something in between: structured enough to provide clarity, simple enough to actually use.

The Solution

A one-page delivery board updated weekly:

Key Features

1. Single Source of Truth

Everything fits on one screen:

  • Current priorities
  • Ownership assignments
  • Risk visibility
  • Timeline awareness

No clicking through tabs or drilling into sub-pages.

2. Weekly Rhythm

Monday Morning Update (15 min):

  1. Review last week’s deliverables
  2. Set this week’s priorities
  3. Update risk status
  4. Identify blockers

Friday Check-in (10 min):

  1. Mark completed items
  2. Note any surprises
  3. Prep for Monday’s planning

3. Risk-First Approach

Risks are prominent, not buried in a separate tab. Each risk includes:

  • Description
  • Impact level (High/Medium/Low)
  • Mitigation strategy
  • Owner responsible for monitoring

4. Clear Ownership

Every deliverable has exactly one owner (not a team). This prevents the “someone else will do it” problem.

State Management (Vanilla JS)

class DeliveryBoard {
  constructor(boardId) {
    this.boardId = boardId;
    this.data = this.loadFromStorage();
    this.render();
  }
  
  loadFromStorage() {
    const stored = localStorage.getItem(`board-${this.boardId}`);
    return stored ? JSON.parse(stored) : this.getDefaultData();
  }
  
  saveToStorage() {
    localStorage.setItem(`board-${this.boardId}`, JSON.stringify(this.data));
  }
  
  toggleTask(taskId) {
    const task = this.data.tasks.find(t => t.id === taskId);
    if (task) {
      task.completed = !task.completed;
      task.completedAt = task.completed ? new Date().toISOString() : null;
      this.saveToStorage();
      this.render();
    }
  }
  
  addRisk(risk) {
    this.data.risks.push({
      id: Date.now(),
      ...risk,
      createdAt: new Date().toISOString()
    });
    this.saveToStorage();
    this.render();
  }
  
  resolveBlocker(blockerId) {
    this.data.blockers = this.data.blockers.filter(b => b.id !== blockerId);
    this.data.resolvedBlockers.push({
      ...this.data.blockers.find(b => b.id === blockerId),
      resolvedAt: new Date().toISOString()
    });
    this.saveToStorage();
    this.render();
  }
  
  render() {
    // Render all sections
    this.renderTasks();
    this.renderRisks();
    this.renderCheckpoints();
    this.renderBlockers();
  }
}

Usage Pattern

Week 1: Setup

  1. Create board for project
  2. Add initial deliverables
  3. Identify known risks
  4. Schedule checkpoints

Week 2-N: Maintain

  1. Monday: Update deliverables, assess risks
  2. Daily: Check off completed tasks
  3. Friday: Review progress, prep for next week

When Project Ends

  1. Archive board
  2. Export data for retrospective
  3. Document lessons learned

Benefits Observed

For Teams:

  • Reduced meeting time (no status update meetings needed)
  • Clearer accountability
  • Earlier risk identification
  • Better stakeholder communication

For Managers:

  • Instant project health visibility
  • No more “surprise” delays
  • Data-driven resource allocation
  • Easier portfolio oversight

Try It Yourself

  3. Start adding your project data
  4. Bookmark the page for daily access

Simple tools used consistently beat complex tools used sporadically. This delivery board proves you don’t need enterprise software to track projects effectively—just clarity, consistency, and commitment to keeping information current.
