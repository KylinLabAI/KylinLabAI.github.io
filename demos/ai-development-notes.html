AI-assisted development notes

Prototype April 28, 2026

Overview

A structured approach to using AI coding assistants that maintains code quality, security, and engineering ownership while accelerating development speed.

The Challenge

AI coding tools (GitHub Copilot, Claude, Cursor, etc.) can generate code incredibly fast, but they introduce new risks:

  • Code you don’t fully understand
  • Security vulnerabilities from training data
  • Inconsistent patterns with your codebase
  • Over-reliance reducing team skills

This demo shows how to harness AI power responsibly.

Framework: The AIR Pattern

Assert → Implement → Review

Phase 1: Assert (Define the Problem)

Before asking AI to code, clearly define:

# Task: User Authentication Module

## Requirements
- Email/password login
- JWT token generation (24h expiry)
- Password hashing with bcrypt
- Rate limiting (5 attempts/minute)
- Session invalidation on logout

## Constraints
- Must use existing PostgreSQL database
- Follow project's error handling patterns
- No external dependencies beyond bcrypt
- TypeScript strict mode enabled

## Success Criteria
- All unit tests passing
- Security audit passes OWASP checklist
- Performance: <100ms authentication time
- Code coverage >85%

## Context
- Existing user model in `models/user.ts`
- Database connection in `db/connection.ts`
- Error handling utility in `utils/errors.ts`

Phase 2: Implement (Generate with Guidance)

Provide AI with specific context and examples:

You are implementing a user authentication module for an Express.js application.

EXISTING CODE PATTERNS:
[Show examples of error handling, async patterns, etc.]

IMPLEMENTATION REQUEST:
Create `services/auth.ts` with:
1. authenticateUser(email, password) function
2. generateToken(user) function  
3. validateToken(token) function
4. logout(userId) function

REQUIREMENTS:
[List requirements from Assert phase]

CONSTRAINTS:
- Use TypeScript
- Follow existing error handling pattern
- Include JSDoc comments
- Add input validation

Phase 3: Review (Human Verification)

Systematic review checklist:

## Code Review Checklist

### Security ✓
- [ ] Password hashing uses bcrypt with salt rounds ≥10
- [ ] No hardcoded secrets or API keys
- [ ] SQL injection prevention (parameterized queries)
- [ ] Rate limiting implemented
- [ ] Token expiration configured
- [ ] Sensitive data not logged

### Quality ✓
- [ ] Code follows project style guide
- [ ] Error messages are user-friendly
- [ ] No console.log in production code
- [ ] Proper async/await usage
- [ ] Type definitions complete
- [ ] Edge cases handled

### Testing ✓
- [ ] Unit tests for all public functions
- [ ] Test coverage >85%
- [ ] Edge cases tested (invalid input, expired tokens)
- [ ] Integration tests for auth flow
- [ ] Security tests (brute force, token tampering)

### Documentation ✓
- [ ] JSDoc comments on public APIs
- [ ] Usage examples in README
- [ ] Security considerations documented
- [ ] Error codes documented

2. Code Review Assistant

Automated preliminary review:

// tools/review-assistant.ts

interface ReviewResult {
  passed: boolean;
  issues: ReviewIssue[];
  suggestions: string[];
}

interface ReviewIssue {
  severity: 'critical' | 'high' | 'medium' | 'low';
  category: 'security' | 'quality' | 'performance' | 'maintainability';
  message: string;
  line?: number;
  suggestion?: string;
}

class ReviewAssistant {
  private rules: ReviewRule[];
  
  constructor() {
    this.rules = this.loadReviewRules();
  }
  
  async reviewCode(code: string): Promise<ReviewResult> {
    const issues: ReviewIssue[] = [];
    
    // Security checks
    issues.push(...this.checkSecurity(code));
    
    // Quality checks
    issues.push(...this.checkQuality(code));
    
    // Performance checks
    issues.push(...this.checkPerformance(code));
    
    return {
      passed: !issues.some(i => i.severity === 'critical'),
      issues,
      suggestions: this.generateSuggestions(issues)
    };
  }
  
  private checkSecurity(code: string): ReviewIssue[] {
    const issues: ReviewIssue[] = [];
    
    // Check for hardcoded secrets
    if (/password\s*=\s*['"][^'"]+['"]/i.test(code)) {
      issues.push({
        severity: 'critical',
        category: 'security',
        message: 'Potential hardcoded password detected',
        suggestion: 'Use environment variables for secrets'
      });
    }
    
    // Check for SQL injection risk
    if (/execute\(\s*[`'"].*\$\{/.test(code)) {
      issues.push({
        severity: 'critical',
        category: 'security',
        message: 'Potential SQL injection vulnerability',
        suggestion: 'Use parameterized queries'
      });
    }
    
    // Check for proper password hashing
    if (!/bcrypt\.hash/.test(code) && /password/.test(code)) {
      issues.push({
        severity: 'high',
        category: 'security',
        message: 'Password may not be properly hashed',
        suggestion: 'Use bcrypt.hash() for password storage'
      });
    }
    
    return issues;
  }
  
  private checkQuality(code: string): ReviewIssue[] {
    const issues: ReviewIssue[] = [];
    
    // Check for console.log
    if (/console\.log/.test(code)) {
      issues.push({
        severity: 'medium',
        category: 'quality',
        message: 'Console.log found in code',
        suggestion: 'Use proper logging framework'
      });
    }
    
    // Check for any type
    if (/: any\b/.test(code)) {
      issues.push({
        severity: 'low',
        category: 'quality',
        message: 'Explicit "any" type used',
        suggestion: 'Use specific types for better type safety'
      });
    }
    
    return issues;
  }
}

3. Implementation Tracker

Track what AI generated vs. human-written:

// tools/implementation-tracker.ts

interface CodeContribution {
  filePath: string;
  aiGenerated: number; // percentage
  humanModified: number; // percentage
  humanWritten: number; // percentage
  reviewNotes: string[];
  timestamp: Date;
}

class ImplementationTracker {
  private contributions: CodeContribution[] = [];
  
  recordContribution(contribution: CodeContribution) {
    this.contributions.push(contribution);
    
    // Log to file for audit trail
    this.saveToLog(contribution);
  }
  
  generateReport(): string {
    const totalFiles = this.contributions.length;
    const avgAiPercentage = this.contributions.reduce(
      (sum, c) => sum + c.aiGenerated, 0
    ) / totalFiles;
    
    return `
AI-Assisted Development Report
===============================
Total files analyzed: ${totalFiles}
Average AI contribution: ${avgAiPercentage.toFixed(1)}%
Average human modification: ${this.calculateAvgModification()}%

High AI-dependency files (>80% AI):
${this.listHighAIFiles()}

Recommendations:
${this.generateRecommendations()}
    `.trim();
  }
  
  private listHighAIFiles(): string {
    return this.contributions
      .filter(c => c.aiGenerated > 80)
      .map(c => `- ${c.filePath} (${c.aiGenerated}% AI)`)
      .join('\n') || 'None';
  }
}

Workflow Demonstration

Step-by-Step Example

Task: Create a password reset feature

1. Assert Phase (5 minutes)

# Password Reset Feature

## User Story
As a user, I want to reset my password via email so I can regain account access.

## Technical Requirements
1. Generate secure reset token (UUID v4)
2. Store token with expiry (1 hour)
3. Send email with reset link
4. Validate token on click
5. Allow password update
6. Invalidate token after use

## Security Considerations
- Token must be cryptographically secure
- Rate limit reset requests (3 per hour)
- Don't reveal if email exists in system
- Force logout all sessions after reset
- Log reset attempts for audit

## Integration Points
- Email service: existing SendGrid integration
- Database: users table + password_resets table
- Frontend: React form at /reset-password

2. Implement Phase (AI generates, 2 minutes)

// services/passwordReset.ts - AI Generated

import { v4 as uuidv4 } from 'uuid';
import bcrypt from 'bcrypt';
import { db } from '../db/connection';
import { sendEmail } from '../services/email';

export async function requestPasswordReset(email: string): Promise<void> {
  // Find user (don't reveal if not found)
  const user = await db.user.findUnique({ where: { email } });
  
  if (!user) {
    // Still return success to prevent email enumeration
    return;
  }
  
  // Check rate limit
  const recentRequests = await db.passwordReset.count({
    where: {
      userId: user.id,
      createdAt: { gt: new Date(Date.now() - 3600000) }
    }
  });
  
  if (recentRequests >= 3) {
    throw new Error('Too many reset requests. Try again later.');
  }
  
  // Generate token
  const token = uuidv4();
  const expiresAt = new Date(Date.now() + 3600000); // 1 hour
  
  // Store token
  await db.passwordReset.create({
    data: {
      userId: user.id,
      token,
      expiresAt,
      used: false
    }
  });
  
  // Send email
  const resetLink = `https://app.example.com/reset-password?token=${token}`;
  await sendEmail({
    to: email,
    subject: 'Password Reset Request',
    html: `<p>Click here to reset your password: <a href="${resetLink}">Reset Password</a></p>`
  });
}

export async function resetPassword(token: string, newPassword: string): Promise<boolean> {
  // Find valid token
  const resetRequest = await db.passwordReset.findFirst({
    where: {
      token,
      used: false,
      expiresAt: { gt: new Date() }
    },
    include: { user: true }
  });
  
  if (!resetRequest) {
    return false;
  }
  
  // Hash new password
  const hashedPassword = await bcrypt.hash(newPassword, 12);
  
  // Update password
  await db.user.update({
    where: { id: resetRequest.userId },
    data: { password: hashedPassword }
  });
  
  // Mark token as used
  await db.passwordReset.update({
    where: { id: resetRequest.id },
    data: { used: true }
  });
  
  // Logout all sessions
  await db.session.deleteMany({
    where: { userId: resetRequest.userId }
  });
  
  return true;
}

// Human improvements applied:

import { v4 as uuidv4 } from 'uuid';
import bcrypt from 'bcrypt';
import { db } from '../db/connection';
import { sendEmail } from '../services/email';
import { logger } from '../utils/logger'; // Added: proper logging
import { RateLimiter } from '../utils/rateLimiter'; // Added: dedicated rate limiter

const RESET_RATE_LIMIT = 3;
const RESET_WINDOW_MS = 3600000; // 1 hour
const BCRYPT_ROUNDS = 12;

/**
 * Request password reset for user
 * @param email - User's email address
 * @throws {Error} If rate limit exceeded
 */
export async function requestPasswordReset(email: string): Promise<void> {
  try {
    // Normalize email
    const normalizedEmail = email.toLowerCase().trim();
    
    // Find user (don't reveal if not found)
    const user = await db.user.findUnique({ 
      where: { email: normalizedEmail } 
    });
    
    // Always return success to prevent email enumeration
    if (!user) {
      logger.info('Password reset requested for non-existent email', { 
        email: normalizedEmail 
      });
      return;
    }
    
    // Check rate limit using dedicated service
    const isRateLimited = await RateLimiter.checkLimit(
      `password-reset:${user.id}`,
      RESET_RATE_LIMIT,
      RESET_WINDOW_MS
    );
    
    if (isRateLimited) {
      logger.warn('Password reset rate limit exceeded', { userId: user.id });
      throw new Error('Too many reset requests. Try again in 1 hour.');
    }
    
    // Generate secure token
    const token = uuidv4();
    const expiresAt = new Date(Date.now() + RESET_WINDOW_MS);
    
    // Store token with transaction
    await db.$transaction([
      db.passwordReset.create({
        data: {
          userId: user.id,
          token,
          expiresAt,
          used: false,
          ipAddress: null, // TODO: Get from request context
          userAgent: null  // TODO: Get from request context
        }
      })
    ]);
    
    // Send email asynchronously (don't block response)
    sendResetEmail(normalizedEmail, token).catch(err => {
      logger.error('Failed to send reset email', { error: err, userId: user.id });
    });
    
    logger.info('Password reset email sent', { userId: user.id });
  } catch (error) {
    logger.error('Password reset request failed', { error, email });
    throw error;
  }
}

/**
 * Reset password using valid token
 * @param token - Reset token from email
 * @param newPassword - New password (must meet complexity requirements)
 * @returns true if successful, false if token invalid
 */
export async function resetPassword(token: string, newPassword: string): Promise<boolean> {
  try {
    // Validate password strength
    if (!isValidPassword(newPassword)) {
      throw new Error('Password does not meet security requirements');
    }
    
    // Find valid token with user
    const resetRequest = await db.passwordReset.findFirst({
      where: {
        token,
        used: false,
        expiresAt: { gt: new Date() }
      },
      include: { user: true }
    });
    
    if (!resetRequest) {
      logger.warn('Invalid or expired reset token used');
      return false;
    }
    
    // Hash new password with appropriate rounds
    const hashedPassword = await bcrypt.hash(newPassword, BCRYPT_ROUNDS);
    
    // Update password and invalidate token in transaction
    await db.$transaction([
      db.user.update({
        where: { id: resetRequest.userId },
        data: { 
          password: hashedPassword,
          updatedAt: new Date()
        }
      }),
      db.passwordReset.update({
        where: { id: resetRequest.id },
        data: { used: true }
      }),
      // Invalidate all active sessions
      db.session.deleteMany({
        where: { userId: resetRequest.userId }
      })
    ]);
    
    logger.info('Password successfully reset', { userId: resetRequest.userId });
    
    // Send confirmation email
    sendPasswordChangedEmail(resetRequest.user.email).catch(err => {
      logger.error('Failed to send password change confirmation', { error: err });
    });
    
    return true;
  } catch (error) {
    logger.error('Password reset failed', { error, token });
    throw error;
  }
}

function isValidPassword(password: string): boolean {
  // Minimum 8 characters, at least one uppercase, one lowercase, one number
  const passwordRegex = /^(?=.*[a-z])(?=.*[A-Z])(?=.*\d).{8,}$/;
  return passwordRegex.test(password);
}

    subject: 'Password Changed Successfully',
    html: '<p>Your password has been changed successfully. If this wasn\'t you, contact support immediately.</p>'
  });
}

Best Practices Summary

Conclusion

AI-assisted development isn’t about replacing developers—it’s about amplifying their capabilities while maintaining control, quality, and security. The AIR pattern (Assert → Implement → Review) provides a structured approach that delivers speed without sacrificing standards.

Use AI as a powerful junior developer who needs careful supervision, not as an autonomous coding oracle.
